


# 1.5 サイト内ルーティング（DC-HQ）

本社（Headquarters）およびデータセンター（Data Center）サイト内の**すべての Layer 3 の非 SD-WAN ルータ間**で、IPv4 サイト内ルーティングのために **OSPF** を構成しなさい。

## 要件

- データセンターおよび本社内の**すべての OSPF ネイバーを正常に確立**しなければならない。
- **R11** と **R12** は、本社内の対向ルータとの OSPF 隣接関係を**MD5 認証**で保護しなければならない。
  - **Key ID:** `1`
  - **暗号化していないキー（平文キー）:** `CCIE!nfr4`

## OSPF を有効化するインターフェース

OSPF は**すべてのインターフェースで有効化**しなければならない。ただし、以下のインターフェースを除く。

- R23 - GE4
- R24 - GE4
- R21 - GE1
- R22 - GE1
- R11 - GE0/0
- R12 - GE0/0

## ルート再配布

- R21 で学習した eBGP ルートを、ディストリビューションスイッチへ再配布（redistribute）しなければならない。
