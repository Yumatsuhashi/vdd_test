# Pattern 2 – Q7（Task 1.7 MPLS アンダーレイ）各コマンドの意味と PE/P 役割

## 質問

pattern2_q7.md（Task 1.7 MPLSアンダーレイ）の解答コマンドについて、
- 各コマンドの意味（何をしているか、問題文のどの部分を満たすために設定しているか）
- PE ルーターか CE ルーターか
を解説してまとめてほしい。

## 回答

### 大前提：ルータの役割

Global SP #1 = プロバイダ網そのもの。R1〜R6 はすべてプロバイダ側で、**CE ルータは1台も含まれない**。

| ルータ | Loopback0 | 役割 | LDP ピア | コア IF |
|---|---|---|---|---|
| R1 | 100.255.254.1 | **P（コア/ハブ）** | R2/R3/R5（3台） | Gi0/0-2（3本） |
| R2 | 100.255.254.2 | **P（コア/ハブ）** | R1/R4/R6（3台） | Gi0/0-2（3本） |
| R3 | 100.255.254.3 | **PE** | R1 のみ | Gi8（1本） |
| R4 | 100.255.254.4 | **PE** | R2 のみ | Gi8（1本） |
| R5 | 100.255.254.5 | **PE** | R1 のみ | Gi8（1本） |
| R6 | 100.255.254.6 | **PE** | R2 のみ | Gi8（1本） |

- R1/R2 = P ルータ（VRF なし。Task 1.8 の VRF fabd2 は R3〜R6 のみ）。コアで複数 LDP ピアを束ねるハブ → グループ（グローバル）認証。
- R3〜R6 = PE ルータ（Branch3/Branch4/SP_2 側に面し VRF fabd2 を持つ）。コア向け LDP ピアが1本 → ピア単位認証で足りる。
- CE（R61/R62/R70/R11/R12 等）は MPLS/LDP を動かさない → 本問のコマンドは CE には一切設定しない。

⚠️ パスワード表記：問題文は `CCIE!nfr4`、解答 config（original）は `CC!E!nfr4`。実機では問題文の正確な文字列を使う。

### A. R1〜R6 共通（OSPF + LDP の土台）

| コマンド | 意味 | 満たす要件 |
|---|---|---|
| `mpls ldp router-id Loopback0 force` | LDP ルータ ID を Loopback0 に固定、force で即時反映 | Loopback0 を LDP ルータ ID として使用 |
| `router ospf 1` | OSPF プロセス起動（コアの IGP） | R1〜R6 に OSPF 設定 |
| `prefix-suppression` | LSA からトランジットリンクの接続プレフィックスを広告しない（Loopback は広告） | LSA にトランジットリンク情報を含めない |
| `mpls ldp autoconfig area 0` | OSPF area 0 全 IF で LDP 自動有効化 | コアでラベルスイッチング設定 |
| `interface Loopback0` / `ip ospf 1 area 0` | Loopback を area 0 で広告（LDP/BGP 到達性の前提） | 内部到達性確保 |
| `interface range Gi0/0-2`（R1/R2）/ `interface Gi8`（R3〜R6） | コア向け物理リンク。R1/R2=3本(ハブ)、R3〜R6=1本(スポーク) | 内部リンク特定 |
| `ip ospf 1 area 0` | コアリンクを area 0 に参加、OSPF ネイバー確立 | すべての内部リンクで OSPF ネイバー確立 |
| `ip ospf network point-to-point` | P2P 化。DR/BDR 選挙なし＝**Type 2 LSA 生成なし** | LSDB に Type 2 LSA が存在しない |
| `mpls ip` | リンクで LDP（ラベルスイッチング）有効化 | ラベルスイッチング設定 |

要点：Type 2 排除 = `ip ospf network point-to-point`、トランジット情報排除 = `prefix-suppression`（2つセットで両要件を満たす）。

### B. Pattern 2 固有：LDP MD5 認証

要件「未承認ピア防止＋偽装 TCP ブロック」= LDP セッション(TCP)の MD5 認証。

B-1. R1・R2（P/ハブ）＝グループ（グローバル）認証
```
access-list 11 permit <peer-Loopback>        # 複数ピアを ACL でグループ化
mpls ldp password option 1 for 11 CC!E!nfr4  # グループ全体に MD5 を一括適用（グローバル）
mpls ldp password required for 11            # そのグループはパスワード必須
```
- R1 の ACL: .2/.3/.5（R2/R3/R5）、R2 の ACL: .1/.4/.6（R1/R4/R6）
- 満たす要件：「R1/R2 は複数ピアグループに LDP MD5 をグローバル設定」「すべてのピアにパスワード必須」

B-2. R3・R4・R5・R6（PE/スポーク）＝ピア単位認証
```
mpls ldp neighbor <peer-ip> password CC!E!nfr4   # 特定1ピアのみ設定
```
- R3→R1(.1)、R4→R2(.2)、R5→R1(.1)、R6→R2(.2)
- 満たす要件：「R3〜R6 は LDP MD5 をピア単位で設定」

### まとめ
- R1/R2 = P：共通＋グループ認証（access-list + password option 1 for + password required）
- R3〜R6 = PE：共通＋ピア単位認証（mpls ldp neighbor … password）
- CE = 設定対象外
- ハブ(3ピア)/スポーク(1ピア) のトポロジ差が、そのまま認証方式（グループ vs ピア単位）の差に対応。
