# Pattern 2 Task 1.5 — SW102 で「OSPF will not operate on this interface until IP is configured on it」が出た件

## 質問

% OSPF will not operate on this interface until IP is configured on it.
SW102(config-if-range)#ip ospf 1 a 0
% OSPF will not operate on this interface until IP is configured on it. CMLの事前設定を見たらこれは普通のエラーなのか？

## 回答

メッセージ自体は IOS の標準的な情報メッセージであり、致命的なエラーではない。「このインターフェースには IP アドレスが設定されていないので、IP が入るまで OSPF はこのインターフェース上で動作しません」という意味で、`ip ospf <process> area <area>` コマンド自体は config に受理される（拒否はされない）。あとから `ip address` を追加すれば、コマンドを打ち直さなくても自動的に OSPF が動き始める。

ただし注目すべきは、EI_v2.yaml の day0（初期）コンフィグ上では、SW102 の Gi0/0・Gi0/1 にはすでに IP アドレスと `ip ospf 1 area 0` の両方が設定済みという点。

```
interface GigabitEthernet0/0
 no switchport
 ip address 10.1.11.2 255.255.255.252   ! R12-Gi0/3 との対向リンク
 ip ospf 1 area 0
!
interface GigabitEthernet0/1
 no switchport
 ip address 10.1.13.2 255.255.255.252   ! R11-Gi0/2 との対向リンク
 ip ospf 1 area 0
```

つまり day0 の設計通りにラボが起動していれば、このコマンドを打ってもこの警告は出ないはず。警告が出たということは、今実際に動いている SW102 の running-config には、day0 で定義されているはずの IP アドレスが（少なくとも Gi0/0-1 のどちらか、あるいは両方）乗っていない状態にある、ということを意味する。

考えられる原因:

1. **day0 コンフィグがロードされずに起動した**: CML でノードが Boot Disk の初期コンフィグを読み込む前にコンソールへ接続してしまった、あるいは day0 の適用に失敗し、工場出荷状態（IP なし）のまま起動している。
2. **ゼロからコンフィグを手打ちしている途中**: day0 を使わず最初から自分で全コマンドを打ち込んでいる場合、`no switchport` → `ip address` をまだ入力していない段階で `ip ospf 1 area 0` を先に打ってしまった。
3. **`no switchport` が未実施**: L3 スイッチのポートはデフォルトで switchport（L2）のままなので、先に `no switchport` をしないと `ip address` コマンド自体が入らない。

**確認方法**:

```
show ip interface brief | include GigabitEthernet0/0|GigabitEthernet0/1
show running-config interface GigabitEthernet0/0
```

IP が無ければ、day0 の値通りに以下を追加すれば解消する（OSPF のコマンド自体は打ち直す必要はない）。

```
interface GigabitEthernet0/0
 no switchport
 ip address 10.1.11.2 255.255.255.252
interface GigabitEthernet0/1
 no switchport
 ip address 10.1.13.2 255.255.255.252
```

`interface range` コマンドは範囲内の各インターフェースに個別にコマンドを適用するため、Gi0/0 と Gi0/1 の両方に IP が無ければ、質問文にある通りこの警告が2回連続で表示される（今回のログはまさにその形）。
