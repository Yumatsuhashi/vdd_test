# Pattern 2 Task 1.5 — R21 の BGP→OSPF 再配布の設計意図（HQ-DC 接続との関係）

## 質問

pattern2 task1.5のsw101においてinterface range GigabitEthernet0/0-1について なぜR21において再配送しているの？ 意図としてはHQとDCを結びつけたいってこどこ？ R21とR11の間にはSW101と201が入っているけどこれらではやらないの？

## 回答

3つの疑問を順番に答える。結論として「HQとDCを結びつけるため」という理解は不正確。HQとDCは Task 1.5 の OSPF area 0 設定（SW101⇔SW201、SW102⇔SW202 のリンク）によって、再配布とは別にすでに1つの OSPF ドメインとして繋がっている。R21 の再配布の目的は別にある。

### 1. なぜ R21 で再配布するのか

DC 内で **SP と直接 eBGP を話しているルータは R21 だけ** だから。DC 内の各ルータの BGP 設定比較：

| デバイス | BGP 設定 |
|---|---|
| **R21** | `router bgp 65002`、`neighbor 100.3.21.1 remote-as 10000`（**eBGP**、対向は SP 領域の R3）＋ `neighbor 10.2.255.22 remote-as 65002`（R22 と iBGP） |
| R22 | `router bgp 65002`、`neighbor 10.2.255.21 remote-as 65002 next-hop-self`（**R21 との iBGP のみ**、外部とは直接話さない） |
| R23 / R24 | BGP 設定なし（OSPF のみ） |
| SW201 / SW202 | BGP 設定なし（OSPF のみ） |

DC は「R21 が SP から eBGP でルートを受け取り、そのルートを OSPF に再配布することで DC 内の全 IGP スピーカーに配る」設計。R22 は R21 から iBGP（next-hop-self 付き）で同じルートをもらう構造なので、R22 自身が再配布する必要はない。

### 2. 意図は「HQ と DC を結びつける」ことか

違う。HQ と DC は Task 1.5 の OSPF 有効化そのもの（SW101⇔SW201、SW102⇔SW202 の L3 リンクが同じ `router ospf 1 area 0` に参加）によって、redistribute とは無関係にすでに1つのルーティングドメインとして接続されている。

R21 の `redistribute bgp 65002 ... subnets` の役割は、「DC が SP から学習した外部ルートを、既存の OSPF ドメイン（DC だけでなく地続きの HQ も含む）へ流し込む」こと。HQ・DC 間の接続性の話ではなく、外部（SP 側）への到達性を社内 IGP 側に伝える話。

HQ 側にも対称的な設計がある。HQ の R11 も SP と eBGP（AS 65001、`neighbor 100.3.11.1 remote-as 10000`）を張っているが、Task 1.5 の時点ではネイバー確立のみで再配布コマンドが入っていない。`redistribute bgp 65001 subnets metric-type 1` が R11 に入るのは別タスク「Task 1.9 — OSPF再配布 / フィルタ」（`original/RS コンフィグ パターン比較.html` 2815〜2823行付近）。「サイトの eBGP 学習ルートを自サイトの OSPF に注入する」という同じパターンが HQ（R11・Task 1.9）と DC（R21・Task 1.5）の両方に存在し、出題タスクが分かれているだけ。

### 3. R21 と R11 の間の SW101・SW201 ではやらないのか

やらない。SW101 や SW201 は BGP を一切話していない（BGP プロセス自体を持たない、OSPF area 0 のみ）ので、そもそも再配布する BGP ルートを持たない。

`redistribute bgp` は「そのルータの BGP テーブルの中身を OSPF プロセスに注入する」コマンドで、BGP を実行していないデバイスでは実行できない。設計原則としても、再配布は外部ドメインとの境界（＝ eBGP を実際に話しているルータ）で1箇所だけ行うのが定石。一度 R21 の OSPF に入れば、OSPF の性質上その経路情報は area 0 内の全デバイス（SW201、SW202、R22、さらにリンク先の SW101/SW102、R11/R12 まで）へ自動的に伝播するため、中継の SW101・SW201 で改めて何かする必要はない。
