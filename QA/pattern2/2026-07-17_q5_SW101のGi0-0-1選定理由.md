# Pattern 2 Task 1.5 — SW101 の interface range GigabitEthernet0/0-1 の選定理由

## 質問

pattern2 task1.5のsw101においてinterface range GigabitEthernet0/0-1について なぜgig0/0-1なの？ほかのインターフェースは？

## 回答

**結論**: `GigabitEthernet0/0-1` が選ばれているのは、この2つのポートが **R11・R12 に直結しているポートそのもの** だからです。Task 1.5 の要件「R11 と R12 は対向ルータとの OSPF 隣接を MD5 認証で保護する」を満たすには、R11/R12 側だけでなく **対向（リンクの反対側）にも同じ MD5 キーを設定しないとネイバーが確立しません**（OSPF の MD5 認証はリンクの両端で鍵が一致している必要があるため）。SW101 から見てその「対向」にあたるのが Gi0/0（→R11）と Gi0/1（→R12）です。

### SW101 のインターフェースとリンク対応

| インターフェース | 接続先 | OSPF 状態（初期コンフィグ） | Task 1.5 での扱い |
|---|---|---|---|
| Gi0/0 | R11 Gi0/3 | area 0 済み | **認証を追加**（対向 R11 が MD5 必須） |
| Gi0/1 | R12 Gi0/2 | area 0 済み | **認証を追加**（対向 R12 が MD5 必須） |
| Gi0/2 | SW201 Gi1/2（DC 側） | area 0 済み | 変更なし（対向が R11/R12 でないため認証不要） |
| Loopback0 | — | area 0 済み | 変更なし |
| Vlan2000 / Vlan2001 | SVI（10.1.100.2/24, 10.1.101.2/24） | 未設定 | **新規追加**（`interface range vlan 2000-2001` で area 0 に参入） |
| Gi1/2, Gi1/3 | SW110 へのトランク | switchport（IP なし） | 対象外（L2 ポートなので OSPF が乗らない） |
| Gi2/0, Gi2/1（Po3 メンバー） | SW102 へのトランク | switchport（IP なし） | 対象外（同上） |
| Port-channel1, Port-channel3 | トランク | switchport（IP なし） | 対象外（同上） |

「すべてのインターフェースで OSPF を有効化」という要件に対して、SW101 で実際に手を加える必要があったのは次の2グループだけです。

1. **Vlan2000/2001**: 初期コンフィグで OSPF が入っていなかったので新規追加（要件で明示的に除外されていないため）。
2. **Gi0/0-1**: 元々 area 0 には入っていたが、対向の R11/R12 側で MD5 認証が必須になったため、リンクの反対側であるここにも同じ認証設定を追加。連番（0 と 1）なので `interface range` でまとめて書けた。

Gi0/2・Loopback0 はすでに要件を満たしていたので触る必要がなく、Gi1/2・Gi1/3・Gi2/0・Gi2/1・Port-channel1/3 はスイッチポート（L2 トランク）で IP アドレスを持たないため、そもそも OSPF（L3 プロトコル）の設定対象になりません。これらのトランクが運ぶ VLAN 2000/2001 の L3 到達性は、SVI である Vlan2000/2001 インターフェース側で OSPF に参加させることでカバーされています。

### 補足: SW102 も対称構成

SW102 は SW101 と逆向きに R11/R12 へ接続しています（Gi0/0→R12、Gi0/1→R11、Gi0/2→SW202）。SW101・SW102 の両方が R11・R12 双方にリンクを持つことで、本社サイト内の冗長性（デュアルホーム）を確保する設計になっています。これが SW102 の解答も SW101 と同一パターン（Gi0/0-1 に認証追加）になっている理由です。
