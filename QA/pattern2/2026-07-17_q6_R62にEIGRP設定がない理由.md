# Pattern2 問題6 — R62にEIGRP設定がない（ように見える）理由

## 質問

q6においてR61ではeigprのaddress-familyとかで設定しているのにR62ではeigrpの設定をしていないのはなぜ？eigrpって片方だけ設定すればいいんだっけ？

## 回答

結論から言うと誤解で、**EIGRPは片方だけ設定すればいいプロトコルではない**。ネイバー隣接（hello交換・認証・トポロジ交換）を成立させるには、リンクの両端で「同じAS番号」「同じ認証方式・キー」「双方がpassiveでない（少なくとも一方が能動的にhelloを出し、相手も受信・応答できる）」という条件が揃っている必要がある。R61だけ設定してR62に何もなければ、Gi0/1（R61 10.6.99.1 ⇔ R62 10.6.99.2）でネイバーは一切成立しない。

### なぜ差分表でR62が「設定なし」に見えるのか

`original/RS コンフィグ パターン比較.html` のTask 1.6表は、CMLラボの**初期コンフィグ（EI_v2.yaml）からの差分**だけを示している。R62の初期コンフィグを確認すると、実は**R61でこれから追加する内容とほぼ同一のEIGRP named mode設定がすでに投入済み**だった:

```ios
router eigrp ccie
 address-family ipv4 unicast autonomous-system 65006
  af-interface default
   passive-interface
  exit-af-interface
  af-interface GigabitEthernet0/1
   authentication mode md5
   authentication key-chain CCIE_MD5
   no passive-interface
  exit-af-interface
  af-interface GigabitEthernet0/2
   authentication mode md5
   authentication key-chain CCIE_MD5
   no passive-interface
  exit-af-interface
  af-interface GigabitEthernet0/3
   authentication mode md5
   authentication key-chain CCIE_MD5
   no passive-interface
  exit-af-interface
  network 10.0.0.0
 exit-address-family
```

つまり「R62に設定していない」のではなく「R62はラボ開始時点ですでにR61と対になる設定が仕込まれている」のが正しい理解。差分表のPattern 1/2列で `router eigrp ccie` ブロックが空欄なのは、これ以上追加する項目がないから。

対してR61の初期コンフィグは `af-interface default / passive-interface` だけで、Gi0/1〜0/3の個別設定もnetwork文もkey chainも無い。だから **Task 1.6の実質的な作業内容はR61側の追加がメイン**になっている——これが「R61だけ設定しているように見える」理由。

### それでもR62に唯一残っている作業

R62の初期コンフィグは `authentication key-chain CCIE_MD5` を参照しているが、その実体となる **`key chain CCIE_MD5`（key-string定義）自体がR62にはまだ存在しない**。認証方式だけ指定してキーチェーンの実体がなければ、EIGRP的には認証情報を積めずhelloが飛ばない（またはネイバー確立に失敗する）。そのため、Task 1.6の解答表でもR62に対して

```ios
key chain CCIE_MD5
 key 1
  key-string CC!E!nfr4
```

の3行だけが Pattern 1/2/3 共通で必要な追加項目として載っている。R61側でも同じkey chainブロックを追加しているので、結果的に**両端で同じキーチェーンが定義され、MD5認証がかみ合う**という構図になる。

まとめると:
- EIGRPは両端の設定が必要（passive/認証/AS番号が揃わないと隣接しない） — これは誤解が正しく修正されるべき点
- R62が「設定なしに見える」のは、CMLラボの初期コンフィグに事前投入されていたから（`ccie-qa` スキルが警告する「差分表に機器名がない＝未設定、ではない」の典型例）
- R62にも実際にはkey chain定義という残作業があり、それはR61と共通で行う
