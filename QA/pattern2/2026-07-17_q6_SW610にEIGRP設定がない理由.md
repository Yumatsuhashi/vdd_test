# Pattern2 問題6 — SW610にEIGRP設定がない理由

## 質問

q6ではbranch3の中でsw610以外の機器でeigrpの設定をしているけど、sw610で設定をしていない理由はなんですか？

## 回答

Branch3の他の機器（R61, R62, SW601, SW602）はすべて**ルーテッドインターフェース（L3）を持つ**のに対し、SW610は**純粋なL2アクセススイッチ**でルーテッドインターフェースを一切持たない。EIGRPはL3ルーティングプロトコルなので、ルーティングしない機器には乗せる場所自体が無い。

### SW610の初期コンフィグ（EI_v2.yaml）

```ios
hostname SW610
vlan 2001
interface GigabitEthernet2/0
 switchport trunk allowed vlan 1,2000
 switchport trunk encapsulation dot1q
 switchport mode trunk
interface GigabitEthernet2/1
 switchport trunk allowed vlan 1,2000
 switchport trunk encapsulation dot1q
 switchport mode trunk
interface Vlan2000
 ip address 10.6.100.10 255.255.255.0
```

SW601・SW602への2本のアップリンク（Gi2/0→SW601, Gi2/1→SW602）は両方とも `switchport mode trunk`。`no switchport`（ルーテッドポート）が1本もない。SW601/SW602の物理ポート（Gi0/0〜0/2）が `no switchport` でL3アドレスを持つのと対照的。

### なぜL3インターフェースが無いと不要なのか

- EIGRPが広報するのは「ルーテッドインターフェースが所属するサブネット」。SW610にはそれが存在しない。
- SW610唯一のIPアドレスはVlan2000のSVI（10.6.100.10/24）だが、これは**SW601・SW602がすでにEIGRPで広報している同一サブネット（10.6.100.0/24）上の1ホストアドレス**にすぎない。新しく広報すべき別のサブネットを持ち込んでいるわけではないので、SW610自身がEIGRPを喋る必要がない。
- L3ゲートウェイ（デフォルトゲートウェイ）はSW601/SW602側のVlan2000 SVIが担っており、HSRPで冗長化されている（`standby 100 ip 10.6.100.1`、SW601側priority 110で優先）。SW610はそのセグメントにぶら下がる下位のアクセススイッチという位置づけ。
- `vlan 2001` の宣言はあるがSW610にはVlan2001のSVIが無い（トランクで素通しするだけ）。つまりVLAN間ルーティングも行っていない＝ルーティング機能そのものを使っていない。

### トポロジ上の役割

topology2.pngで見ると、SW610はSW601とSW602の間にぶら下がる形（Gi2/0→SW601, Gi2/1→SW602のクロス配線）で、下流にPC61・PC62を収容するアクセス層スイッチ。SW601⇄SW610⇄SW602という冗長トランク構成はSTP（PVST）でループを防ぎつつ、L3側の冗長性はSW601/SW602のHSRPペアで担う——典型的な「アクセス層はL2でシンプルに、L3ゲートウェイ機能はディストリビューション層（SW601/SW602）に集約する」設計。アクセス層スイッチにルーティングプロトコルを乗せないのはCCIEに限らず一般的なベストプラクティスでもある（管理を複雑にせず、ルーティングドメインを不必要に広げない）。
