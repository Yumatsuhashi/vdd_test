# Pattern 2 Task 1.5 — R23/R24 で router-id を明示している理由

## 質問

R23と24で
router ospf 1
router-id 10.2.255.23

しているのはなぜ？他のルーターではしていないじゃん

## 回答

まず事実として、router-id の明示が必要なタイミングは機器によって違う。

| デバイス | router-id 明示のタイミング |
|---|---|
| **R23 / R24** | **Pattern 1・2・3 すべて**で必須（`router ospf 1` / `router-id 10.2.255.23` などが row-common） |
| R21 / R22 / SW201 / SW202 / SW211 / SW212 | **Pattern 3 のみ**の追加（verify セクションにも明記: 「Pattern 3 追加（Router-ID / Timers）...SW201/SW202/SW211/SW212/R21/R22 等にrouter-id追加」） |

つまり「他のルータではしていない」というより正確には、R21/R22 は Pattern 3 で初めて router-id を明示する設計。Task 1.5（Pattern 1・2）の時点では R21・R22 に router-id コマンドは出てこない。

### なぜ R21 は Pattern 1・2 で不要なのか（確度: 高）

R21 は day0（CML の初期コンフィグ）の時点で、すでに `router ospf 1` の中に `redistribute bgp 65002 subnets` が入っており、`interface Loopback0` にも `ip ospf 1 area 0` が焼き込み済み。つまりラボが起動した瞬間から R21 の OSPF プロセスは Loopback0 を area 0 メンバーとして認識している。OSPF の router-id 自動選出は「稼働中の loopback インターフェースの中で一番高い IP（存在すれば無条件に物理インターフェースより優先）」というルールなので、R21 は最初から迷いようがなく `10.2.255.21` に決まる。だから明示指定が不要。

### なぜ R23 / R24 は Pattern 1 から必須なのか（確度: 中、推測を含む）

R23・R24 は day0 に `router ospf 1` 自体が一切存在せず、OSPF をゼロから作る。この点は R22 も同様（day0 に空の `router ospf 1` はあるが Loopback0 には OSPF 未設定）で、教科書的な自動選出ルール通りなら R22 も R23 も Loopback の IP が正しく自動選出されるはず。実際 R22 は Pattern 1・2 で明示不要になっており、これは自動選出が期待通り機能している証拠と言える。

一方 R23・R24 に固有な点として、Gi1 が SP（SP_1）へ直結する WAN 向けインターフェースで、そこに `200.99.23.2`（R23）/ `200.99.24.2`（R24 相当）という Loopback（`10.2.255.2x`）より数値の大きい IP が乗っている（`no shutdown` で up 状態）。教科書ルール上は「loopback が存在すれば数値に関わらず必ず loopback が勝つ」はずだが、CCIE のラボ運用では OSPF プロセス生成・インターフェース投入の順序やタイミング（`interface range GigabitEthernet2-3, Loopback0` は Gi2→Gi3→Loopback0 の順で適用される）によって router-id が意図しない値に固定されてしまうリスクを避けるため、「迷う可能性がある機器では最初から明示する」という設計思想が採られている可能性が高い。R23・R24 はさらに Pattern 2 でデフォルトルート（`ip route 0.0.0.0 0.0.0.0 200.99.23.1`）、Pattern 3 で OSPF タイマー調整（`timers pacing flood`, `queue-depth update`）と、他パターンでも継続的に個別チューニングされる SP 対向ルータであるため、検証コマンド（`show ip ospf | include Router ID`）で毎回確実な値を返す必要があった、というのが最も説得力のある実務的理由と考えられる。

ただし、この「なぜ R23/R24 だけ最初から必須か」という設計意図そのものは original 資料に明文化されておらず、上記は IOS の router-id 選出ルールとトポロジ上の役割から導いた推測。R21 が Pattern 1/2 で不要な理由（day0 に OSPF が焼き込み済みだから）ほどの確証はない点に留意。
