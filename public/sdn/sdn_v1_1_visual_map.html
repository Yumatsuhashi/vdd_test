Template の仕組みと関係性

SD-WAN Template 階層の考え方と、各タスクで作る/編集するテンプレートの意味

📐 Template の基本構造

SD-WAN の Template は 2層構造：

Device Template (DT)
＝
Feature Template (FT)
×
複数個の組み合わせ
  • Device Template (DT) = デバイス1台に対応する「設定の入れ物」。Attach するのはこれ
  • Feature Template (FT) = VPN設定・Interface設定・OSPF設定など個々の機能パーツ
  • Device Specific 変数 = 同じ DT を複数台に使い回す時、Attach 時にデバイスごとに入力

⚠️ 「Global」 vs 「Device Specific」 vs 「Default」

  • Global — 全デバイス共通の固定値（例: VPN ID=999, OSPF Area 0, Telnet ON）
  • Device Specific — Attach 時にデバイスごとに入力（例: IP, Hostname, IF名）→ 暗記必須
  • Default — Cisco デフォルト値をそのまま使用

💡 BR1-cEdge40-Global の「Global」は上記の設定モードとは別物 — これは FT の名前。Telnet Outband を Global (= 固定ON) に設定する FT。

🔑 v1.1 の VPN-999-Template は DC と Branch で共有

DC cEdge21/22 も Branch cEdge40 も 同じ VPN-999-Template を使う。物理 IF が違う（DC=Gi2/Gi3, Branch=Gi4）のに使い回すため：

  • Parent IF Template が必要 — Sub-IF (Gi4.3999) の前に物理 IF (Gi4) を定義
  • IF 名は Device Specific 変数 → Attach 時にデバイスごとに入力
  • Variable List に vpn999_if_name_parent = GigabitEthernet4 が出る
v1.3 との違い: v1.3 は Branch 専用の CCIE-BR1-VPN999-Template を新規作成 → Parent IF 不要 / vpn999_if_name_parent も出ない

🔑 VPN999 の OSPF は Branch 内だけじゃない

OSPF = cEdge ↔ ローカル SW の接続手段。学習した経路は OMP 経由で全サイトに配布される。

🔑 OMP vs BGP — 使い分け

  • OMP — サイト間の 全 経路交換（VPN999 経路, TLOC, Policy）。DTLS トンネル内で動作
  • BGP — overlay の 外側 だけ（DC Underlay: R22↔cEdge / v1.3 の TLOC Extension: ISP 戻り経路）

Branch 間は BGP ではなく OMP。BGP は SD-WAN overlay に入る「前」と「後」の物理ルーティング用。

🔑 Color = WAN 回線のラベル

private1 private2 default biz-internet は物理 WAN 回線を識別するタグ。同じ color 同士で IPsec トンネル (TLOC) が張れる。

  • SP ルータ自体は SD-WAN から透過的 — cEdge の WAN IF に color を設定するだけ
  • v1.1 Branch: default / v1.3 Branch: private1 — 試験で混同注意
  • AAR の Preferred Color でトラフィックを特定の WAN に誘導
